# Constraint Programming-Guided Reinforcement Learning in the Stochastic FrozenLake Environment

## Project Description

This project investigates the effectiveness of using Constraint Programming (CP) and Belief Propagation (BP) to generate
reward shaping signals for Q-learning agents operating in the stochastic FrozenLake-v1 environment. The primary
objective is to enhance learning efficiency and final policy quality in scenarios characterized by sparse rewards and
uncertain state transitions.

Two novel CP-based reward shaping methods are introduced and evaluated:

* **CP-Marginal Strength (CP-MS) Shaping:** Leverages action marginal probabilities derived from the CP model.
* **CP-Expected Total Reward (CP-ETR) Shaping:** Uses the CP model's estimate of the expected total future reward.

These methods are systematically compared against several baselines:

* Standard Q-learning (no shaping)
* Q-learning with classic potential-based shaping (Manhattan distance)
* A CP-MS Greedy heuristic agent (a non-learning, planning-based approach, implemented without caching for these
  results)

The evaluation is conducted across various 4x4 and 8x8 slippery FrozenLake instances, designed to offer a range of
complexities.

## Repository Contents

* **`main.py`**: The main script to configure and run experiments.
* **`environment.py`**: Handles creation and management of FrozenLake environment instances.
* **`q_learning_standard.py`**: Implements standard Q-learning and Q-learning with classic potential-based shaping.
* **`q_learning_cp.py`**: Implements Q-learning agents with CP-MS and CP-ETR reward shaping, including the client for
  communication with the CP server and caching mechanisms for shaped rewards/underlying CP values.
* **`heuristic_agents.py`**: Implements the CP-MS Greedy agent (non-cached version for reported results) and an Optimal
  policy agent.
* **`config.py`**: Stores hyperparameters for Q-learning algorithms and evaluation parameters.
* **`utils.py`**: Contains utility functions for saving/loading results, agent evaluation, and plotting learning curves.
* **`instances.json`**: Defines configurations for different FrozenLake instances (e.g., size, hole patterns, custom
  maps, CP model parameters).
* **`MiniCPBP/`**: Directory containing the Java-based Constraint Programming server (utilizing MiniCPBP) that provides
  marginal probabilities and expected total reward values to the Python agents. It includes a `pom.xml` for Maven build
  and execution.
* **`report.ipynb`**: A Jupyter notebook used for orchestrating batch experiments, aggregating results from multiple
  runs/seeds, and generating comparative plots.
* **`notebook_plots/`**: (Typically generated by `report.ipynb` or other plotting scripts) Directory where plots, such
  as learning curves and map layouts, are saved.
* **`results/`**: (Generated by `main.py`) Directory where raw experimental data, including JSON log files and
  Q-tables (CSV), are stored.

## How to Run

### Prerequisites

* Python 3 (e.g., 3.9+)
* Required Python packages: `numpy`, `gymnasium` (version 1.1.1 or as used in `environment.py`), `matplotlib`. Install
  via pip:
    ```bash
    pip install numpy gymnasium matplotlib
    ```
* Java Development Kit (JDK) (e.g., JDK 11 or compatible).
* Apache Maven (for compiling and running the `MiniCPBP` Java server). Ensure `mvn` is in your system's PATH.

### Running an Experiment

Experiments are launched using the `main.py` script. For CP-based agents, this script will automatically attempt to
compile (if necessary) and run the `MiniCPBP` Java server.

**General Command Structure:**

```bash
python main.py --instance <INSTANCE_ID> --agent <AGENT_TYPE> --shaping <SHAPING_METHOD> --episodes <NUM_EPISODES> --seed <SEED_VALUE>
```

#### Key Arguments:

* `--instance <INSTANCE_ID>`: Required. The ID of the FrozenLake instance as defined in `instances.json` (e.g., `4s`,
  `4medium`, `4hard`, `8s`, `8medium`, `8hard`).
* `--agent <AGENT_TYPE>`: Specifies the agent to run.
    * `q`: Q-learning agent. Must be accompanied by a `--shaping` argument.
    * `cp_greedy`: The CP-MS Greedy heuristic agent (uses `minicpbp.examples.FrozenLakeCPService`).
    * `optimal`: The agent following a pre-defined optimal policy (if available in `instances.json`).
* `--shaping <SHAPING_METHOD>`: Required if `--agent q`. Defines the reward shaping technique.
    * `none`: Standard Q-learning (no shaping).
    * `classic`: Q-learning with classic potential-based shaping.
    * `cp-ms`: Q-learning with CP-Marginal Strength shaping (uses `minicpbp.examples.FrozenLakeCPServiceMS`).
    * `cp-etr`: Q-learning with CP-Expected Total Reward shaping (uses `minicpbp.examples.FrozenLakeCPServiceETR`).
* `--episodes <NUM_EPISODES>`: Number of training episodes for Q-learning agents (e.g., 500). Default is 500. Ignored
  for non-learning agents.
* `--seed <SEED_VALUE>`: Integer for the random seed to ensure reproducibility (default is taken from `config.py`).

#### Example Commands:

Run Q-learning with CP-ETR shaping on instance `4hard` for 500 episodes with seed 1:

```bash
python main.py --instance 4hard --agent q --shaping cp-etr --episodes 500 --seed 1
```

Run the CP-MS Greedy agent (non-cached version) on instance `4s` (evaluation episodes defined in `config.py`, seed 1):

```bash
python main.py --instance 4s --agent cp_greedy --seed 1
```

The appropriate MiniCPBP Java service will be started from the `MiniCPBP/` directory based on the agent/shaping method
selected. Experimental outputs (logs, Q-tables) are saved in the `results/` directory. Plots generated by `report.ipynb`
or individual runs are typically found in `notebook_plots/` or `plots/`.

## Key Findings (Summary)

* **CP-ETR Shaping for Q-learning:** Generally proves to be the most robust and effective learning method among those
  tested, achieving stable convergence to near-optimal policies on several 4x4 and 8x8 instances.
* **CP-MS Greedy Heuristic (Non-Cached):** This non-learning agent often demonstrates optimal or very close to optimal
  performance, highlighting the strong guidance potential directly extractable from the CP-BP model at each step. It
  serves as a powerful benchmark.
* **CP-MS Shaping for Q-learning:** Can accelerate initial learning but sometimes converges to suboptimal policies,
  particularly on more complex maps. Its shaping signal, not being strictly potential-based, might sometimes misguide
  long-term optimization.
* **Classic Potential-Based Shaping:** Shows unreliable and mixed performance. While effective on some specific
  instances (e.g., `4medium`, `8hard`), the simplistic nature of the Manhattan distance heuristic limits its general
  applicability in complex layouts.
* **Standard Q-Learning:** Performs poorly across all tested instances within the 500-episode training limit,
  underscoring the difficulty of learning in these sparse and stochastic environments without effective guidance.
* **Computational Considerations:** CP-based methods (both shaping for Q-learning and the greedy heuristic) exhibit good
  sample efficiency in terms of episodes required for learning/performance. However, they incur a higher wall-clock time
  per episode due to the overhead of communication with the Java CP server and the CP-BP computation (one BP iteration
  per query in this project). Caching strategies for shaped rewards and underlying CP-derived values were implemented in
  the Q-learning agents to mitigate redundant queries (though the reported CP-MS Greedy results are from a non-cached
  version).

This project demonstrates that integrating lookahead information from CP-BP models, particularly through the CP-ETR
shaping mechanism, offers a promising approach to guide Q-learning agents effectively in stochastic grid-world
environments, though the practical application must consider the associated computational trade-offs.
